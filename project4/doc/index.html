["Read a Poem Before Bed" — An Algorithmic Poetry Generator and Reader]

yudi zhang | PoetryDB API work with Google Cloud Text-to-Speech API | project 4 | fall 2024

What I Made

I wanted to explore how machines and humans can work together creatively. I created a simple webpage where users can read and listen to poems before bed. The webpage lets users:

  • Generate a random poem.
  • Search for poems using a keyword.
  • Listen to the poem using a text-to-speech feature.
  • Save the poem as a text file.

How I Made It

Generating Poems

  • I used the PoetryDB API to fetch poems. Users can type a keyword to find poems related to it, or they can just click a button to generate a random one. I only show the first 5 lines of a poem to keep it short and easy to read.

code that fetches the poems:

const baseUrl = 'https://poetrydb.org'; const apiUrl = keyword ? `${baseUrl}/lines/${keyword}` : `${baseUrl}/random`; const response = await fetch(apiUrl); const poems = await response.json(); const filteredPoems = poems.filter(poem => poem.lines.some(line => line.toLowerCase().includes(keyword.toLowerCase())) );

Adding Text-to-Speech

  • I used the Google Cloud Text-to-Speech API to make the poems "speak." When users click the "Read It" button, the tool converts the poem into audio. The audio is then played directly in the browser.

code of how I made it work:

const requestPayload = { input: { text: poemText }, voice: { languageCode: 'en-US', ssmlGender: 'NEUTRAL' }, audioConfig: { audioEncoding: 'MP3' } }; const audioResponse = await fetch(url, { method: 'POST', body: JSON.stringify(requestPayload) }); const audioData = await audioResponse.json();

Adding a Save Option

  • I added a button that lets users save the generated poem as a .txt file. This is done using JavaScript's Blob API.
